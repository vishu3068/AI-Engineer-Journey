# Day 13 – Abstraction

## What is Abstraction?

Abstraction means hiding implementation details and showing only the required functionality to the user.

## Why use Abstraction?

It hides complex implementation, improves code organization, and ensures important methods are implemented by child classes.

## ABC

ABC stands for Abstract Base Class.

## Abstract Class

An abstract class is a blueprint that cannot be instantiated directly.

## Abstract Method

An abstract method has no implementation in the parent class and must be implemented in child classes.

## @abstractmethod

Used to declare an abstract method.

## Difference

Encapsulation hides data.

Abstraction hides implementation.
